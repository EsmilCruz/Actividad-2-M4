Respuestas a Preguntas sobre Modelos de Estimación y Desarrollo de Software


1. ¿Qué son los Modelos de Estimación?

Los modelos de estimación son técnicas y métodos utilizados para predecir el esfuerzo, tiempo y costo necesarios para completar un proyecto de software. Estos modelos ayudan a los equipos de desarrollo a planificar y gestionar recursos de manera efectiva.


2. Investigación sobre Diferentes Modelos de Estimación

Modelo COCOMO (COnstructive COst MOdel)

Ventajas: Proporciona estimaciones basadas en el tamaño del software en líneas de código, considerando diferentes niveles de proyecto (orgánico, semiacoplado, embebido).

Desventajas: Requiere datos históricos precisos y puede ser complejo de aplicar en proyectos pequeños o ágiles.

Situaciones Adecuadas: Grandes proyectos de software con datos históricos disponibles.


Método de Puntos de Función

Ventajas: Se basa en las funcionalidades del software desde la perspectiva del usuario, no depende del lenguaje de programación o tecnología utilizada.

Desventajas: Puede ser subjetivo y requiere experiencia para aplicar correctamente.

Situaciones Adecuadas: Proyectos donde la funcionalidad del usuario es más relevante que el tamaño del código.


3. ¿A qué llamamos la estimación de un proyecto de software?

La estimación de un proyecto de software es el proceso de prever la cantidad de esfuerzo, tiempo y recursos necesarios para completar el proyecto de desarrollo de software.


4. Principios Esenciales para el Desarrollo de Sistemas

Modularidad: Dividir un sistema en módulos o componentes independientes.

Cohesión: Medida en la que los elementos de un módulo están relacionados y trabajan juntos.

Acoplamiento: Grado de interdependencia entre los módulos; debe ser bajo.

Abstracción: Ocultar detalles complejos y mostrar sólo lo esencial.


Ejemplo de Aplicación: Modularidad

En un sistema de gestión de inventario, se podría aplicar la modularidad separando las funciones de gestión de productos, control de inventarios y generación de reportes en módulos independientes.


5. Partes Interesadas en un Proyecto de Desarrollo de Software para una Aplicación Móvil

Clientes: Definen los requisitos y esperan un producto que cumpla sus necesidades.

Desarrolladores: Implementan el software, responsables de la codificación y pruebas.

Gerentes de Proyecto: Planifican y supervisan el proyecto, responsables de la entrega a tiempo y dentro del presupuesto.

Usuarios Finales: Utilizan la aplicación, esperan una interfaz fácil de usar y funcionalidad eficiente.

Financistas: Proveen los fondos necesarios, esperan un retorno de inversión.

Testers/QA: Aseguran la calidad del software mediante pruebas exhaustivas.

7. Normas y Estándares para el Desarrollo de Software

Consistencia en Codificación: Utilizar nombres de variables en camelCase.

Documentación Clara y Concisa: Documentar cada función con comentarios que expliquen su propósito y parámetros.

Revisión de Código Regular: Realizar revisiones de código por pares antes de fusionar cambios en el repositorio principal.

Pruebas Unitarias y de Integración: Escribir pruebas unitarias para cada módulo y pruebas de integración para las interacciones entre módulos.

Seguimiento de Versiones: Utilizar control de versiones como Git para gestionar cambios en el código.
